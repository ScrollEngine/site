Scroll is a blog engine/CMS designed for and built in Node.js. It is meant to be easy to set up and use, but fully customizable so you can tailor it to your needs without headaches.

Design Philosophy

The core philosophy of Scroll is that, in general, is it is unopinionated. It is more of a manager that allows you to use the proven tools and techniques you know and love when customizing your site, while at the same time providing a base for utilizing such things. That is, it does not require a specific client-side framework, view engine, or data store.

This power comes from the fact that Scroll is a strict MVC application, but with each peice being modular. While you have to have a view layer, controller layer, and a model layer, they are each replacable and/or extendable. To make this process easier, Scroll utilizes the proven technology of NPM to manage the model and view layer, so using a different one is as easy as npm installing a different module.

AS you read this guide, you will also notice that there is no fancy CLI commands or custom setup for using Scroll. This is because Scroll is designed to not only be easy to personalize, but also easy to publish. It is PaaS friendly, so to speak. Your Scroll server starts just like any other Node.js application, without the need for proprietary, globally installed commands or generators.

Using

Step one is to NPM install the scroll-core module, which can be considered the controller layer..in a way. It is also the server and manager that handles the model and view layers. By default Scroll will use the scroll-model-mongodb module for the model layer and the scroll-view-basic module for the view layer.

Using and starting the Scroll server is simliar to Express, if you are familiar with that. It only requires a few lines of code to get it started up.

var scroll = require('scroll-core'),
    app = scroll({model:{connection:'[mongo_url]'}});

app.start(9876);

As stated above, this will start a Scroll server, connecting to the MongoDB url given. This will also use the basic view (kind of like a theme), which is not the most amazing ever. So now its all about customizing your Scroll site.

Customizing

Making your Scroll site look and work the way you want can be as easy as installing a different module or as complex as reimplementing most of the Scroll server. It is completely up to you how far you want to go.

View and Model modules

The simpliest way to customize Scroll is to install different view and model layers. You can search for ones that suit your infrastrcutre/design needs then use npm install [module] --save to install and save it to your application.

A model module will, basically, represent how your data is stored and retrieved, an interface for raw data. Most of the time the only important factor is the data store used, IE MongoDB, MySQL, etc.

View modules are your complete front end. They can be thought of themes, but in general they will do much more than just apply some CSS and define pages. They not only manage the view engine behind your front end, but also can define custom routes, manage all your public files, and can even customize how your Scroll application actually works. Just make sure you choose the right view layer that is right for you.

Configuration

When you create your Scroll server, you can pass in a configuration which can override a good amount of functionality of Scroll. See the full list for all the options, but here are some common things you may want to override. See the usage example above for where exactly in the process you provide this configuration.

View Module {view:'scroll-view-basic'}

This must be set in order to use a custom view module.

Model Module {model:{module:'scroll-view-mongodb'}}

This must be set in order to use a custom model module.

Model Connection {model:{connection:'[url]'}}

Set the connection to the data store.

View Pages {views:{post:{view:'anotherView'}}}

This allows you to override the views used in the default
